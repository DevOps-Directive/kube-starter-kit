Extensions are components not included in the base kit but that you may want to add as your team and requirements grow.

<Note>
This section is under development. Check back for guides on adding these capabilities to your platform.
</Note>

## Planned Extensions

### Security & Compliance

| Extension | Use Case |
|-----------|----------|
| **Security/Compliance Baseline** | Pod security standards, network policies, audit logging for SOC2/HIPAA readiness |
| **Multi-tenancy/Namespace Isolation** | Resource quotas, network policies, RBAC patterns for shared clusters |
| **Private Kubernetes Endpoints** | Restrict API server access to private networks with bastion host access patterns |
| **Vault** | Advanced secrets management with dynamic secrets |

### Developer Experience

| Extension | Use Case |
|-----------|----------|
| **Developer Experience Tooling** | Local dev environment parity (Tilt, Telepresence), streamlined "add a new service" workflows |
| **PR Preview Environments** | Ephemeral environments for each pull request to enable testing before merge |

### Deployment & GitOps

| Extension | Use Case |
|-----------|----------|
| **Centralized ArgoCD** | Single ArgoCD instance managing multiple clusters, enables ApplicationSets for dynamic app generation |
| **Kargo** | Environment promotion workflows beyond linear staging/production (canary, blue-green, multi-stage) |
| **Timoni** | Next-generation application management using CUE for type-safe Kubernetes configurations |

### Operations & Reliability

| Extension | Use Case |
|-----------|----------|
| **Cost Visibility** | Kubecost or OpenCost integration, resource defaults, spot instance configuration for AWS spend tracking |
| **EKS Upgrade Automation** | Guidance and tooling for navigating EKS version upgrades |
| **Disaster Recovery/Backup** | Velero for cluster state backup, database backup patterns, restore procedures |
| **Multi-region Setup** | Geographic redundancy and disaster recovery |

### Observability & Networking

| Extension | Use Case |
|-----------|----------|
| **Service Mesh (Istio)** | mTLS, advanced traffic management, observability |
| **Datadog Integration** | Enterprise observability with APM |

### Infrastructure & Data

| Extension | Use Case |
|-----------|----------|
| **PlanetScale / RDS** | Managed database alternatives to CloudNativePG |
| **VPC Peering** | Connect to other AWS accounts or on-premise |

Have a specific extension you'd like to see? [Open an issue](https://github.com/DevOps-Directive/kube-starter-kit/issues) on GitHub.
